# Day Counter PRD

## 1. 개요

- **앱 설명**: 광고 없이 깔끔하고 예쁜 디자인의 D-Day & 기념일 카운터 웹앱
- **해결하는 문제**: 기존 D-Day 앱(TheDayBefore, Dreamdays 등)의 과도한 광고와 구식 디자인에 대한 사용자 불만을 해소한다. 홈 화면에 놓고 싶을 만큼 예쁜 UI와 마일스톤 시각화를 제공하여 중요한 날을 의미 있게 관리할 수 있게 한다.
- **타겟 사용자**:
  - 기념일, 시험, 여행 등 중요한 날짜를 관리하고 싶은 20~30대
  - 깔끔한 디자인을 선호하며 광고에 거부감이 있는 사용자
  - 모바일 웹 브라우저에서 간편하게 사용하고 싶은 사용자

---

## 2. 핵심 기능

### 2.1 D-Day 생성 (P0 - 필수)

- **설명**: 사용자가 새로운 D-Day를 생성한다. 제목, 날짜, 카테고리, 이모지 아이콘을 선택할 수 있다.
- **유저 스토리**: "사용자로서, 중요한 날짜를 제목/카테고리/이모지와 함께 등록하고 싶다. 그래서 한눈에 어떤 날인지 파악할 수 있다."
- **수용 기준**:
  - 제목 입력 필드가 존재하며, 1~30자까지 입력 가능하다.
  - 날짜 선택기(date picker)를 통해 과거/미래 날짜를 선택할 수 있다.
  - 카테고리 5종(기념일, 시험, 여행, 생일, 커스텀) 중 하나를 선택할 수 있다.
  - 카테고리별 추천 이모지 8~10개 중 하나를 선택할 수 있다.
  - "저장" 버튼 클릭 시 유효성 검사 후 저장되고 메인 페이지로 이동한다.
  - 제목 또는 날짜가 비어 있으면 저장할 수 없다.

### 2.2 D-Day 카운트 (P0 - 필수)

- **설명**: 등록된 D-Day까지의 남은 일수(D-30, D-7 등) 또는 지난 일수(D+100 등)를 자동 계산하여 표시한다.
- **유저 스토리**: "사용자로서, 등록한 날짜까지 며칠 남았는지/지났는지 한눈에 보고 싶다. 그래서 일정을 효과적으로 준비할 수 있다."
- **수용 기준**:
  - 미래 날짜는 "D-N" 형태로 남은 일수를 표시한다.
  - 당일은 "D-Day"로 표시한다.
  - 과거 날짜는 "D+N" 형태로 지난 일수를 표시한다.
  - 날짜 계산은 클라이언트 사이드에서 실시간으로 처리한다.
  - 자정이 지나면 자동으로 카운트가 갱신된다.

### 2.3 카드형 리스트 (P0 - 필수)

- **설명**: 등록된 D-Day들을 카테고리별 컬러 그라데이션이 적용된 카드 형태로 리스트에 표시한다.
- **유저 스토리**: "사용자로서, 등록한 D-Day들을 예쁜 카드 목록으로 보고 싶다. 그래서 한눈에 모든 일정을 파악할 수 있다."
- **수용 기준**:
  - 각 D-Day가 개별 카드로 표시된다.
  - 카드에는 이모지, 제목, 카테고리, 남은/지난 일수, 대상 날짜가 표시된다.
  - 카테고리별 다른 그라데이션 배경이 적용된다.
  - 가장 가까운 D-Day가 상단에 하이라이트로 크게 표시된다.
  - 카드 클릭 시 상세 페이지로 이동한다.
  - D-Day가 없을 때 빈 상태 안내 메시지가 표시된다.

### 2.4 상세 보기 (P0 - 필수)

- **설명**: 선택한 D-Day의 상세 정보와 마일스톤 타임라인을 보여주고, 수정/삭제 기능을 제공한다.
- **유저 스토리**: "사용자로서, 특정 D-Day의 상세 정보와 마일스톤을 보고 싶다. 그래서 중요한 시점들을 미리 파악할 수 있다."
- **수용 기준**:
  - 이모지, 제목, 카테고리, 대상 날짜, 남은/지난 일수가 표시된다.
  - 마일스톤 타임라인(D-100, D-50, D-30, D-10, D-7, D-3, D-1, D-Day)이 시각화된다.
  - 이미 지난 마일스톤은 완료 표시, 아직 안 온 마일스톤은 미완료 표시된다.
  - 수정 버튼 클릭 시 편집 모드로 전환된다.
  - 삭제 버튼 클릭 시 확인 다이얼로그 후 삭제되고 메인으로 이동한다.

### 2.5 마일스톤 타임라인 시각화 (P1 - 중요)

- **설명**: D-Day까지의 주요 마일스톤을 시각적 타임라인으로 표시한다.
- **유저 스토리**: "사용자로서, D-Day까지의 주요 시점들을 타임라인으로 보고 싶다. 그래서 진행 상황을 직관적으로 파악할 수 있다."
- **수용 기준**:
  - D-100, D-50, D-30, D-10, D-7, D-3, D-1, D-Day 마일스톤이 세로 타임라인으로 표시된다.
  - 각 마일스톤에 날짜와 남은 일수가 함께 표시된다.
  - 지난 마일스톤은 체크 아이콘 + 완료 스타일로 표시된다.
  - 현재 가장 가까운 마일스톤이 강조 표시된다.
  - 대상 날짜가 100일 미만일 경우 D-100, D-50 등은 자동으로 제외된다.

### 2.6 정렬/필터 (P1 - 중요)

- **설명**: 메인 리스트에서 카테고리별 필터링과 날짜순/이름순 정렬 기능을 제공한다.
- **유저 스토리**: "사용자로서, D-Day 목록을 카테고리로 필터링하거나 정렬하고 싶다. 그래서 원하는 일정을 빠르게 찾을 수 있다."
- **수용 기준**:
  - 카테고리 필터 칩(전체/기념일/시험/여행/생일/커스텀)이 가로 스크롤로 표시된다.
  - 칩 클릭 시 해당 카테고리만 필터링된다.
  - 정렬 옵션: 날짜 가까운 순(기본), 날짜 먼 순, 이름순, 생성순
  - 필터와 정렬 상태는 페이지 새로고침 시 초기화된다.

### 2.7 다크모드 (P1 - 중요)

- **설명**: 시스템 설정을 자동 감지하고 수동 토글도 지원하는 다크모드를 제공한다.
- **유저 스토리**: "사용자로서, 어두운 환경에서도 편안하게 앱을 사용하고 싶다. 그래서 다크모드를 켤 수 있다."
- **수용 기준**:
  - 시스템 다크모드 설정을 자동으로 감지한다.
  - 설정 페이지에서 수동으로 라이트/다크/시스템 모드를 전환할 수 있다.
  - 다크모드 설정은 로컬스토리지에 저장되어 유지된다.
  - 모든 페이지와 컴포넌트가 다크모드에 대응한다.

## 3. 페이지 구성

### 3.1 메인 페이지 (/)

- **URL 경로**: `/`
- **포함 컴포넌트**:
  - `Header`: 앱 로고/타이틀 + 설정 아이콘
  - `HeroCard`: 가장 가까운 D-Day를 크게 하이라이트 표시
  - `CategoryFilter`: 카테고리 필터 칩 바
  - `SortSelector`: 정렬 옵션 드롭다운
  - `DdayCardList`: D-Day 카드 그리드/리스트
  - `DdayCard`: 개별 D-Day 카드
  - `EmptyState`: D-Day가 없을 때 안내 화면
  - `FloatingAddButton`: 우하단 고정 추가 버튼 (FAB)
- **사용자 흐름**:
  1. 앱 진입 시 등록된 D-Day 목록이 카드 형태로 표시된다.
  2. 상단에 가장 가까운 D-Day가 하이라이트로 크게 보인다.
  3. 카테고리 칩을 탭하여 필터링할 수 있다.
  4. 카드를 탭하면 상세 페이지로 이동한다.
  5. 우하단 "+" 버튼을 탭하면 추가 페이지로 이동한다.
  6. D-Day가 없으면 빈 상태 안내와 함께 추가 유도 버튼이 보인다.

### 3.2 D-Day 추가 페이지 (/new)

- **URL 경로**: `/new`
- **포함 컴포넌트**:
  - `PageHeader`: 뒤로가기 + 페이지 제목
  - `TitleInput`: 제목 입력 필드
  - `DatePicker`: 날짜 선택기 (HTML date input 기반)
  - `CategorySelector`: 카테고리 선택 칩
  - `EmojiPicker`: 카테고리별 추천 이모지 선택 그리드
  - `SaveButton`: 저장 버튼
- **사용자 흐름**:
  1. 제목을 입력한다.
  2. 날짜를 선택한다.
  3. 카테고리를 선택하면 해당 카테고리의 추천 이모지가 표시된다.
  4. 이모지를 선택한다. (선택하지 않으면 카테고리 기본 이모지 적용)
  5. "저장" 버튼을 탭하면 유효성 검사 후 저장, 메인으로 이동한다.

### 3.3 D-Day 상세 페이지 (/detail/[id])

- **URL 경로**: `/detail/[id]`
- **포함 컴포넌트**:
  - `PageHeader`: 뒤로가기 + 수정/삭제 액션 버튼
  - `DetailHero`: 이모지 + 제목 + 카운트 대형 표시
  - `DateInfo`: 대상 날짜 + 등록일 정보
  - `MilestoneTimeline`: 마일스톤 세로 타임라인
  - `EditModal`: 수정 모달 (인라인 편집)
  - `DeleteConfirmDialog`: 삭제 확인 다이얼로그
- **사용자 흐름**:
  1. 카드에서 진입하면 상세 정보가 큰 화면으로 표시된다.
  2. 마일스톤 타임라인을 스크롤하며 확인할 수 있다.
  3. 상단 수정 아이콘 탭 시 편집 모달이 열린다.
  4. 삭제 아이콘 탭 시 확인 다이얼로그가 뜨고, 확인 시 삭제 후 메인으로 이동한다.

### 3.4 설정 페이지 (/settings)

- **URL 경로**: `/settings`
- **포함 컴포넌트**:
  - `PageHeader`: 뒤로가기 + 페이지 제목
  - `ThemeToggle`: 라이트/다크/시스템 모드 선택
  - `DataManagement`: 전체 삭제 + 데이터 내보내기/가져오기
  - `AppInfo`: 앱 버전, 제작 정보
- **사용자 흐름**:
  1. 다크모드를 라이트/다크/시스템 중 선택한다.
  2. 전체 데이터 삭제를 할 수 있다. (확인 다이얼로그 필수)
  3. 앱 정보를 확인할 수 있다.

## 4. 비기능 요구사항

### 반응형 디자인
- **모바일 (< 640px)**: 카드 1열 배치, 풀 와이드 레이아웃
- **태블릿 (640px ~ 1024px)**: 카드 2열 그리드
- **데스크톱 (> 1024px)**: 카드 3열 그리드, 최대 너비 960px 중앙 정렬

### 다크모드 지원
- 시스템 설정 자동 감지 (prefers-color-scheme)
- 수동 토글 (라이트/다크/시스템)
- 로컬스토리지에 설정 저장

### 접근성
- 시맨틱 HTML 사용
- 모든 인터랙티브 요소에 키보드 접근 가능
- 적절한 aria-label 적용
- 색상 대비 WCAG AA 기준 충족

### 성능
- 페이지 초기 로딩 2초 이내
- 날짜 계산은 클라이언트에서 처리 (서버 부하 없음)
- 이미지/외부 리소스 최소화

## 5. 배포 환경

- **호스팅**: Vercel (GitHub 연동 자동 배포)
- **DB**: 프로토타입 단계에서는 로컬스토리지를 1차 저장소로 사용. 로컬 개발 시 SQLite + Prisma로 API 테스트 가능. Vercel 배포 시에는 로컬스토리지 기반으로 동작.
- **외부 API 의존성**: 없음 (외부 API에 의존하지 않는 독립 실행형 앱)
- **데이터 저장 전략**:
  - MVP: 브라우저 로컬스토리지 (서버리스 환경 호환)
  - 확장 시: Vercel Postgres 또는 Supabase로 마이그레이션

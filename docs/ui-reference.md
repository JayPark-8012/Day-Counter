# UI 참조

## 전체 레이아웃 구조

```
<html>
  <body class="bg-white dark:bg-slate-900 text-slate-900 dark:text-slate-100">
    <main class="max-w-3xl mx-auto px-4 py-6 sm:px-6 sm:py-8 min-h-screen">
      {/* 페이지 콘텐츠 */}
    </main>
  </body>
</html>
```

- 모든 페이지는 `max-w-3xl` (768px) 중앙 정렬
- 하단 네비게이션 바 없음 (각 페이지에서 뒤로가기로 이동)
- FAB(추가 버튼)은 메인 페이지에서만 표시

## 1. 메인 페이지 (/)

### 레이아웃

```
+------------------------------------------+
|  Header                                   |
|  [Day Counter 로고]        [설정 아이콘]   |
+------------------------------------------+
|                                           |
|  Hero Card (가장 가까운 D-Day)             |
|  +--------------------------------------+ |
|  |  [이모지]                             | |
|  |  제목                                 | |
|  |  D-25                    (큰 숫자)    | |
|  |  2026년 3월 25일           카테고리    | |
|  +--------------------------------------+ |
|                                           |
|  Category Filter                          |
|  [전체] [기념일] [시험] [여행] [생일] ...  |
|                                           |
|  Sort Selector                            |
|  N개의 D-Day  [날짜 가까운 순 v]          |
|                                           |
|  D-Day Card Grid                          |
|  +------------------+ +------------------+|
|  | [emoji]          | | [emoji]          ||
|  | 제목             | | 제목             ||
|  | D-45       날짜  | | D+120      날짜  ||
|  +------------------+ +------------------+|
|  +------------------+ +------------------+|
|  | ...              | | ...              ||
|  +------------------+ +------------------+|
|                                           |
|                              [+ FAB]      |
+------------------------------------------+
```

### 반응형 동작
- **모바일 (< 640px)**: 카드 1열, Hero Card 풀 와이드
- **태블릿 (640px~)**: 카드 2열 그리드
- **데스크톱 (1024px~)**: 카드 3열 그리드

### 컴포넌트 배치

1. **Header**
   - 위치: 페이지 최상단
   - 좌측: 앱 이름 "Day Counter" (text-xl font-bold)
   - 우측: 설정 아이콘 (Settings, 클릭 시 /settings 이동)
   - 하단 간격: mb-6

2. **HeroCard**
   - 위치: Header 아래
   - 가장 가까운 미래 D-Day를 크게 표시 (미래 D-Day가 없으면 가장 최근 과거 D-Day)
   - D-Day가 하나도 없으면 숨김 처리
   - 카테고리 그라데이션 배경 (진한 버전)
   - 이모지: text-4xl, 제목: text-xl font-bold, 카운트: text-5xl font-extrabold
   - 클릭 시 해당 D-Day 상세 페이지로 이동
   - 하단 간격: mb-6

3. **CategoryFilter**
   - 위치: HeroCard 아래
   - 가로 스크롤 가능한 칩 바 (overflow-x-auto, scrollbar-hide)
   - 칩: 전체(기본 선택) / 기념일 / 시험 / 여행 / 생일 / 커스텀
   - 선택된 칩: bg-indigo-500 text-white
   - 미선택 칩: bg-slate-100 text-slate-600
   - 하단 간격: mb-4

4. **SortSelector**
   - 위치: CategoryFilter 아래
   - 좌측: "N개의 D-Day" 텍스트 (text-sm text-slate-500)
   - 우측: 정렬 드롭다운 또는 토글 버튼
   - 정렬 옵션: 날짜 가까운 순(기본) / 날짜 먼 순 / 이름순 / 생성순
   - 하단 간격: mb-4

5. **DdayCardList**
   - 위치: SortSelector 아래
   - 그리드 레이아웃: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`
   - 하단에 FAB와 겹치지 않도록 `pb-20` 패딩

6. **DdayCard**
   - 카테고리별 그라데이션 배경
   - 상단 좌측: 이모지 (text-3xl)
   - 상단 우측: 카테고리 태그 (text-xs, 카테고리 색상)
   - 중앙: D-Day 카운트 (text-3xl font-extrabold)
   - 하단 좌측: 제목 (text-base font-semibold)
   - 하단 우측: 대상 날짜 (text-sm text-slate-500)
   - 전체 클릭 시 /detail/[id] 이동

7. **EmptyState** (D-Day 없을 때)
   - 중앙 정렬
   - CalendarPlus 아이콘 (text-5xl, text-slate-300)
   - "아직 등록된 D-Day가 없어요" 텍스트
   - "첫 번째 D-Day를 추가해보세요" 부가 텍스트
   - "D-Day 추가하기" 버튼 (Primary 스타일, /new 이동)

8. **FloatingAddButton (FAB)**
   - 위치: 우하단 고정 (fixed bottom-6 right-6)
   - 원형 버튼, Plus 아이콘
   - 클릭 시 /new 이동
   - D-Day가 없어도 항상 표시

### 인터랙션
- 카드 호버 시: 살짝 위로 이동 + 쉐도우 증가
- 필터 칩 클릭 시: 선택 상태 토글, 카드 목록 즉시 필터링 (애니메이션 없이 즉시 반영)
- 정렬 변경 시: 카드 목록 즉시 재정렬
- 빈 상태에서 추가 버튼 클릭 시: /new 이동

## 2. D-Day 추가 페이지 (/new)

### 레이아웃

```
+------------------------------------------+
|  PageHeader                               |
|  [< 뒤로]               D-Day 추가        |
+------------------------------------------+
|                                           |
|  제목                                     |
|  +--------------------------------------+ |
|  |  제목을 입력하세요                     | |
|  +--------------------------------------+ |
|                                           |
|  날짜                                     |
|  +--------------------------------------+ |
|  |  [캘린더 아이콘]  2026-03-25          | |
|  +--------------------------------------+ |
|                                           |
|  카테고리                                 |
|  [기념일] [시험] [여행] [생일] [커스텀]    |
|                                           |
|  이모지                                   |
|  +--------------------------------------+ |
|  |  [em1] [em2] [em3] [em4] [em5]       | |
|  |  [em6] [em7] [em8] [em9] [em10]      | |
|  +--------------------------------------+ |
|                                           |
|  미리보기                                  |
|  +--------------------------------------+ |
|  |  (작성 중인 D-Day 카드 미리보기)       | |
|  +--------------------------------------+ |
|                                           |
|  +--------------------------------------+ |
|  |            저장하기                    | |
|  +--------------------------------------+ |
|                                           |
+------------------------------------------+
```

### 컴포넌트 배치

1. **PageHeader**
   - 좌측: ChevronLeft 아이콘 (클릭 시 뒤로가기)
   - 중앙: "D-Day 추가" 텍스트 (text-lg font-semibold)
   - 하단 간격: mb-8

2. **TitleInput**
   - 라벨: "제목" (text-sm font-medium text-slate-700, mb-2)
   - 입력 필드: 텍스트 input, placeholder="D-Day 제목을 입력하세요"
   - 최대 30자, 글자 수 카운터 표시 (우측 하단, text-xs)
   - 유효성 에러 시 border-red-500 + 에러 메시지 표시
   - 하단 간격: mb-6

3. **DatePicker**
   - 라벨: "날짜" (text-sm font-medium text-slate-700, mb-2)
   - HTML date input (type="date")
   - Calendar 아이콘 좌측 표시
   - 하단 간격: mb-6

4. **CategorySelector**
   - 라벨: "카테고리" (text-sm font-medium text-slate-700, mb-3)
   - 5개 카테고리 칩을 가로 배치 (flex flex-wrap gap-2)
   - 각 칩: 카테고리 이름 + 기본 이모지
   - 선택 시 카테고리 액센트 컬러 적용
   - 하단 간격: mb-6

5. **EmojiPicker**
   - 라벨: "이모지" (text-sm font-medium text-slate-700, mb-3)
   - 선택된 카테고리에 따라 추천 이모지 8~10개 표시
   - 그리드 배치: `grid grid-cols-5 gap-3`
   - 각 이모지: text-2xl, 원형 배경, 클릭 시 선택 상태
   - 선택된 이모지: ring-2 ring-indigo-500 bg-indigo-50
   - 하단 간격: mb-6

6. **미리보기 카드** (선택사항)
   - 작성 중인 정보로 DdayCard 미리보기 표시
   - 실시간 반영 (제목/날짜/카테고리/이모지 변경 시 즉시 갱신)
   - 하단 간격: mb-8

7. **SaveButton**
   - 풀 와이드 Primary 버튼
   - "저장하기" 텍스트
   - 제목/날짜 미입력 시 disabled 상태 (opacity-50, cursor-not-allowed)
   - 저장 중: "저장 중..." + 로딩 스피너

### 인터랙션
- 카테고리 선택 시: 해당 카테고리 추천 이모지로 즉시 갱신, 이전 선택 초기화
- 이모지 미선택 시: 카테고리 기본 이모지가 자동 선택됨
- 저장 클릭 시: 유효성 검사 -> 로컬스토리지 저장 -> 메인 페이지로 이동
- 뒤로가기 시: 작성 중인 내용이 있으면 확인 다이얼로그 표시 (선택적)

### 레이아웃

```
+------------------------------------------+
|  PageHeader                               |

|  +--------------------------------------+ |

```

### 컴포넌트 배치

1. **PageHeader**
   - 좌측: ChevronLeft (뒤로가기, 메인으로 이동)
   - 우측: Pencil 아이콘 (수정) + Trash2 아이콘 (삭제)
   - 하단 간격: mb-6

2. **DetailHero**
   - 카테고리 그라데이션 배경 (rounded-2xl, p-8)
   - 중앙 정렬 (text-center)
   - 이모지: text-6xl, mb-4
   - 제목: text-xl font-bold, mb-6
   - D-Day 카운트: text-6xl font-extrabold tracking-tight, mb-4
     - 미래: "D-25" 형태, text-indigo-600
     - 당일: "D-Day" 형태, text-amber-500 + 반짝이는 효과
     - 과거: "D+100" 형태, text-slate-500
   - 대상 날짜: text-base text-slate-600, "2026년 3월 25일 (화)"
   - 카테고리 태그: 작은 칩 형태
   - 하단 간격: mb-8

3. **MilestoneTimeline**
   - 섹션 제목: "마일스톤" (text-lg font-semibold, mb-4)
   - 세로 타임라인 (좌측 라인 + 원형 포인트)
   - 마일스톤 목록: D-100, D-50, D-30, D-10, D-7, D-3, D-1, D-Day
   - 각 마일스톤 항목:
     - 좌측: 원형 포인트 (지남=인디고 채움, 미지남=회색 테두리, 현재=인디고+pulse)
     - 중앙: 라벨 (D-100) + 날짜 (2025.12.15)
     - 우측: 상태 텍스트 ("지남" / "N일 남음" / "오늘")
   - 대상 날짜가 100일 미만이면 D-100 제외, 50일 미만이면 D-50도 제외 (자동 조정)
   - 하단 간격: mb-6

4. **DateInfo**
   - 등록일 표시 (text-sm text-slate-400)
   - 마지막 수정일 표시 (수정된 경우만)

5. **EditModal** (수정 버튼 클릭 시)
   - 오버레이 + 모달 (max-w-md)
   - 기존 정보가 채워진 폼 (TitleInput + DatePicker + CategorySelector + EmojiPicker)
   - "수정하기" 버튼 + "취소" 버튼
   - 수정 완료 시 모달 닫힘 + 상세 정보 갱신

6. **DeleteConfirmDialog** (삭제 버튼 클릭 시)
   - 오버레이 + 작은 다이얼로그 (max-w-sm)
   - "정말 삭제하시겠습니까?" 메시지
   - "삭제" 버튼 (Danger 스타일) + "취소" 버튼
   - 삭제 확인 시 메인 페이지로 이동

### 인터랙션
- 수정 아이콘 클릭: EditModal 열림
- 삭제 아이콘 클릭: DeleteConfirmDialog 열림
- 수정 저장 시: 모달 닫힘 + 페이지 데이터 즉시 갱신
- 삭제 확인 시: 로컬스토리지에서 삭제 + "/" 로 이동
- 뒤로가기: "/" 로 이동

## 4. 설정 페이지 (/settings)

### 레이아웃

```
+------------------------------------------+
|  PageHeader                               |
|  [< 뒤로]                       설정      |
+------------------------------------------+
|                                           |
|  테마                                     |
|  +--------------------------------------+ |
|  |  [Sun] 라이트  [Moon] 다크  [Mon] 시스템| |
|  +--------------------------------------+ |
|                                           |
|  데이터 관리                               |
|  +--------------------------------------+ |
|  |  [Download] 데이터 내보내기      [>]  | |
|  +--------------------------------------+ |
|  +--------------------------------------+ |
|  |  [Upload] 데이터 가져오기        [>]  | |
|  +--------------------------------------+ |
|  +--------------------------------------+ |
|  |  [Trash2] 모든 데이터 삭제       [>]  | |
|  +--------------------------------------+ |
|                                           |
|  앱 정보                                  |
|  +--------------------------------------+ |
|  |  Day Counter v1.0.0                   | |
|  |  Made with Next.js                    | |
|  +--------------------------------------+ |
|                                           |
+------------------------------------------+
```

### 컴포넌트 배치

1. **PageHeader**
   - 좌측: ChevronLeft (뒤로가기, 메인으로)
   - 중앙: "설정" (text-lg font-semibold)
   - 하단 간격: mb-8

2. **ThemeToggle**
   - 섹션 제목: "테마" (text-base font-semibold text-slate-900, mb-3)
   - 3개 옵션을 세그먼트 컨트롤 형태로 배치
   - 옵션: 라이트 (Sun 아이콘) / 다크 (Moon 아이콘) / 시스템 (Monitor 아이콘)
   - 선택된 옵션: bg-white shadow-sm (라이트), bg-slate-700 (다크) 등 시각적 구분
   - 배경: bg-slate-100 dark:bg-slate-800 rounded-xl p-1
   - 즉시 적용 (클릭과 동시에 테마 변경)
   - 하단 간격: mb-8

3. **DataManagement**
   - 섹션 제목: "데이터 관리" (text-base font-semibold, mb-3)
   - 리스트 형태 (rounded-xl border 내부)
   - 각 항목: 좌측 아이콘 + 텍스트, 우측 화살표
   - 항목 사이 divider (border-b)
   - **데이터 내보내기**: JSON 파일 다운로드
   - **데이터 가져오기**: JSON 파일 업로드 (file input)
   - **모든 데이터 삭제**: 텍스트 빨간색, 클릭 시 확인 다이얼로그
   - 하단 간격: mb-8

4. **AppInfo**
   - 섹션 제목: "앱 정보" (text-base font-semibold, mb-3)
   - 앱 이름, 버전, 기술 스택 등 간략 표시
   - text-sm text-slate-400
   - 중앙 정렬

### 인터랙션
- 테마 토글: 즉시 적용, 로컬스토리지에 저장
- 데이터 내보내기: 클릭 시 JSON 파일 즉시 다운로드
- 데이터 가져오기: 파일 선택 -> 유효성 검사 -> 기존 데이터에 병합/덮어쓰기 선택
- 전체 삭제: 확인 다이얼로그 -> "모든 D-Day가 삭제됩니다. 이 작업은 되돌릴 수 없습니다." -> 삭제 실행 -> 토스트 메시지

## 공통 컴포넌트

### Toast / Snackbar
- 화면 하단 중앙에 일시적으로 표시
- 성공: 에메랄드 배경 + Check 아이콘
- 에러: 레드 배경 + X 아이콘
- 3초 후 자동 사라짐
- `fixed bottom-20 left-1/2 -translate-x-1/2`

### Loading State
- 페이지 로딩: 중앙에 스피너 (animate-spin)
- 버튼 로딩: 버튼 내 텍스트 대신 스피너 표시
- 카드 로딩: 스켈레톤 UI (animate-pulse bg-slate-200)

### Error State
- API 에러 시: 화면 중앙에 에러 메시지 + 재시도 버튼
- 폼 유효성 에러: 해당 필드 하단에 빨간색 에러 텍스트

### 빈 상태 (Empty State)
- 큰 아이콘 + 제목 + 설명 + CTA 버튼
- 중앙 정렬, 부드러운 색상

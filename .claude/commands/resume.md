# /resume 커맨드

세션이 끊긴 후 작업을 이어서 진행하는 커맨드이다.

---

## 실행 순서

### 1. 현재 상태 파악
- `docs/PROGRESS.md`를 읽고 진행 상태를 파악한다
- `git log --oneline -5`로 최근 커밋 이력을 확인한다
- `git status`로 커밋되지 않은 변경사항을 확인한다

### 2. 미완료 작업 확인
PROGRESS.md에서 `[ ]` (미완료) 항목을 찾는다.

### 3. 커밋되지 않은 변경사항 처리
`git status`에서 변경사항이 있으면:
```bash
git add -A
git commit -m "[Resume] 이전 세션 미커밋 변경사항 저장"
```

### 4. 이어서 진행
미완료 Phase의 첫 번째 미완료 항목부터 작업을 재개한다.

- Phase 1 미완료 → planner 서브에이전트에게 위임 (누락된 문서만 생성)
- Phase 2 미완료 → 프로젝트 초기화 이어서 진행
- Phase 3 미완료 → backend-dev 서브에이전트에게 위임 (미구현 API만)
- Phase 4 미완료 → frontend-dev 서브에이전트에게 위임 (미구현 페이지만)
- Phase 5 미완료 → 빌드 및 Git Push 진행
- Phase 6 미완료 → 배포 확인 및 안내

### 5. 블로커 확인
PROGRESS.md의 "블로커" 섹션에 내용이 있으면:
- 블로커 내용을 사용자에게 먼저 알린다
- 해결 방법을 제안한다
- 사용자 확인 후 진행한다

### 6. 완료 보고

> 🔄 작업을 이어서 진행했습니다.
>
> 이전 상태: [Phase X]의 [작업명]까지 완료
> 현재 상태: [Phase Y]의 [작업명]까지 완료
>
> 남은 작업:
> [미완료 항목 목록]

---

## 주의사항
- 이미 생성된 파일은 덮어쓰지 않는다
- 수정이 필요한 파일만 수정한다
- 각 단계 완료 시 PROGRESS.md를 반드시 업데이트한다
- Phase 완료 시 git commit & push도 수행한다
